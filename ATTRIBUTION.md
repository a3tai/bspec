# Attribution and License Policy

This repository contains two distinct classes of content:

- **Specification documentation and templates** (`spec/` and related docs)
- **Code, scripts, SDKs, and tooling**

**Current policy**

- Specification documentation is intended to be released under **Creative Commons Attribution-ShareAlike 4.0 (CC BY-SA 4.0)**.
- Tooling, code, and SDK implementation files continue to follow the repository `LICENSE` terms.
- `docs/ATTRIBUTION.md` is the framework-level attribution index for borrowed external methods.

### Trademark and Framework Rules

- If a trademarked framework name is used (for example, Net Promoter, Stage-Gate, COBIT, etc.), include the legal form in the first mention where practical and set:
  - `attribution_required: true` in frontmatter
  - `source_frameworks` with the framework/rights holder entry
  - any required trademark/legal notice near first mention
- Keep framework attributions consistent across dependent documents and relationship templates.

For detailed framework-by-framework entries, notices, and examples, see [`docs/ATTRIBUTION.md`](docs/ATTRIBUTION.md).

## BSpec Name Clearance

The phrase **BSpec** is a product-standard name and must be treated as a brand identifier pending legal review.

Before using BSpec as a marketed product, certification, or training label:

- Complete USPTO TESS and EUIPO checks in the target jurisdiction(s).
- Confirm there are no blocking prior marks in relevant classes.
- Capture clearance date, registrar record, and legal sign-off in repo records.
- If clearance is not yet complete, use neutral naming in public materials.
