# Business Model

The Business Model domain describes how your organization creates, delivers, and captures value. These documents define revenue streams, pricing, partnerships, and cost structures.

## Document Types (9)

| Code | Name | Description |
|------|------|-------------|
| [CHN](/docs/types/CHN) | Channel Strategy | Distribution and sales channels |
| [CST](/docs/types/CST) | Cost Structure | Major cost categories and drivers |
| [CUS](/docs/types/CUS) | Customer Relationships | How you acquire and retain customers |
| [KAC](/docs/types/KAC) | Key Activities | Core activities required for success |
| [KPT](/docs/types/KPT) | Key Partnerships | Strategic partners and alliances |
| [KRS](/docs/types/KRS) | Key Resources | Critical assets and capabilities |
| [PRI](/docs/types/PRI) | Pricing Strategy | Pricing models and tactics |
| [REV](/docs/types/REV) | Revenue Model | How you generate revenue |
| [VST](/docs/types/VST) | Value Stream | Value creation and delivery flow |

## Key Relationships

Business Model documents typically:
- **Depend on**: Strategy, Product, and Customer insights
- **Enable**: Financial planning and Operations execution

## Getting Started

For Bronze conformance, start with:
1. **REV** - Define your revenue model
2. **PRI** - Set your pricing strategy
