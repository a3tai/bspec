# Customer Value

The Customer Value domain focuses on understanding your customers and the value you create for them. These documents capture customer insights, pain points, journeys, and the gains you deliver.

## Document Types (12)

| Code | Name | Description |
|------|------|-------------|
| [BEH](/docs/types/BEH) | Behaviors | Customer behavioral patterns and habits |
| [CIN](/docs/types/CIN) | Interviews | Customer interview findings and insights |
| [CJM](/docs/types/CJM) | Customer Journey Map | End-to-end customer experience mapping |
| [EMP](/docs/types/EMP) | Empathy Maps | Customer thoughts, feelings, and motivations |
| [FEE](/docs/types/FEE) | Feedback | Customer feedback collection and analysis |
| [GAI](/docs/types/GAI) | Gains | Benefits and outcomes customers seek |
| [JTB](/docs/types/JTB) | Jobs-to-be-Done | Functional and emotional jobs customers hire you for |
| [PAI](/docs/types/PAI) | Pain Points | Customer problems and frustrations |
| [PER](/docs/types/PER) | Personas | Representative customer archetypes |
| [STO](/docs/types/STO) | User Stories | Feature requirements from user perspective |
| [SUR](/docs/types/SUR) | Surveys | Quantitative customer research |
| [USE](/docs/types/USE) | Use Cases | Specific scenarios of product/service usage |

## Key Relationships

Customer Value documents typically:
- **Depend on**: Market Segments and Strategic direction
- **Enable**: Product Requirements, Features, and Business Model

## Getting Started

For Bronze conformance, start with:
1. **PER** - Create customer personas
2. **USE** - Define key use cases
